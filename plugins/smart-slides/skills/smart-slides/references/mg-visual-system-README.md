# Hermes MG 视觉系统样本库

这个目录用于筛选 Hermes 长视频的 MG/HTML 信息层设计，不是生产渲染代码。

## 当前产物

- `mg-template-browser.html`：自包含动态 HTML，可直接打开。
- `MG-VISUAL-STANDARD-v1.md`：MG 视觉标准，定义画面分区、信息层级、信息块规则、布局规则、动画规则和阻断 QC。
- 内置 8 个公开参考源、10 个长视频信息场景、40 个基础动画模块族。
- 页面运行时生成 2000 个模块变体和 500 个模板候选。
- 每个模板包含适用场景、模块组成、信息层数量、时间轴、导演规则、数据结构和 QC 规则。

## 设计原则

1. 不复制外部模板和素材，只抽象表达规律。
2. LLM 后续只负责选择模板、填内容、生成参数。
3. 渲染层应该根据结构化 spec 确定性生成透明 HTML/SVG/CSS。
4. HyperFrames 保留为最终合成和时间轴执行层，不作为自由设计来源。
5. 旧模板或通用卡片只作为 fallback。

## 第一阶段目标

第一阶段先用这个文档做人工筛选：删除不可用模板，标记可用模板，再把可用模板沉淀成 Hermes 的 `mg_design_spec_v1`。

第一批只把 5 个母版按视觉标准重建到可人工筛选的质量线：

- `impact_ripple`：事件中心 + 外圈影响层。
- `evidence_dashboard`：主证据 + 两个辅助证据块。
- `timeline_ridge`：等距时间轴 + 上下交错节点。
- `route_arc`：路径主线 + 起终点 + 流向标签。
- `quote_document`：文档主体 + 扫描高亮 + 来源标签。

这 5 个通过后再扩展到剩余 15 个蓝图和 500 个模板；不能用碰撞 QC 代替人工视觉筛选。

## 推荐筛选标准

- 画面语义是否清楚。
- 是否适合 B-roll 透明叠加。
- 是否能支持 1-5 个信息层。
- 是否能从导演文档稳定选择。
- 是否容易做确定性 QC：安全区、文字溢出、透明背景、时间轴完整。
