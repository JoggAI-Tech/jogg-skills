# MG 视觉标准 v1

这个标准用于 Hermes 长视频的 B-roll + 透明 HTML 信息层。目标不是生成更多样式，而是保证每个 MG 画面有明确的信息层级、视觉路径和可生产的结构。

## 1. 画面分区

每个 16:9 画面固定分为四个区域：

- 左侧叙事区：标题和简短解释，宽度约 38%。
- 右侧主视觉区：动画主体，宽度约 44%。
- 辅助信息区：来源、补充标签、次要节点，只能放在主视觉外圈。
- 底部安全区：保留给 B-roll/字幕/播放器环境，不放核心信息。

主视觉区必须有一个明确主体。不能把多个小卡片随机散落在画面中。

## 2. 信息层级

每个画面最多 5 层信息，但视觉上只允许 3 个层级：

- L1 主视觉：数字、事件、路径、时间轴、文档、仪表等，必须最醒目。
- L2 解释节点：原因、证据、影响、阶段、流向等，最多 2-4 个。
- L3 来源/补充：来源、备注、辅助说明，必须弱于 L2。

同一层级的信息块必须同尺寸、同样式、同对齐规则。上层级不能比下层级更小、更弱。

## 3. 信息块规则

信息块不是默认组件。只有当它承载结构化事实时才使用。

允许：

- 关键节点标签。
- 证据/来源短标签。
- 路径端点。
- 阶段节点。
- 风险阈值。

禁止：

- 为了填空而放卡片。
- 卡片压住主视觉。
- 同屏出现多套卡片样式。
- 卡片只展示一句口播碎片。
- 大量半透明卡片漂浮在无结构区域。

## 4. 布局规则

信息块必须跟随主视觉的结构排列：

- 时间线：节点必须沿时间轴等距排列。
- 路径：起点/终点必须贴近路径两端，但不能压住路径。
- 涟漪：事件在中心，影响节点在外圈，不允许漂到标题区。
- 仪表：阈值/预警在仪表上方或两侧，不压住仪表。
- 文档：来源标签在文档外侧或底部，不压住扫描区域。

每个母版必须先定义视觉骨架，再放信息块。

## 5. 动画规则

动画顺序固定：

1. 主视觉骨架出现。
2. L1 信息进入。
3. L2 信息按口播节奏进入。
4. L3 来源/补充最后进入。
5. 结尾保持可读状态。

不能所有元素同时出现，也不能动效循环抢主信息。

## 6. QC 规则

阻断项：

- 信息块互相重叠。
- 信息块遮挡主视觉。
- 同层级信息块尺寸不一致。
- L1 信息比 L2/L3 更弱。
- 主视觉没有明确主体。
- 文本越界或压住安全区。
- 画面看起来像随机卡片拼贴。

允许项：

- 装饰底板和其承载的信息块重叠。
- 动效轨迹穿过空白区域。
- 低优先级辅助线在背景中弱化存在。

## 7. 第一批标准母版

先按这个标准重做 5 个母版：

- `impact_ripple`：事件中心 + 外圈影响层。
- `evidence_dashboard`：主证据 + 两个辅助证据块。
- `timeline_ridge`：等距时间轴 + 上下交错节点。
- `route_arc`：路径主线 + 起终点 + 流向标签。
- `quote_document`：文档主体 + 扫描高亮 + 来源标签。

这 5 个通过人工视觉筛选后，再扩展到 20 个蓝图和 500 个模板。
